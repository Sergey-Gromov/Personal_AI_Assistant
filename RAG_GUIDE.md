# 📚 RAG (База знаний) - Полное руководство

Руководство по работе с системой RAG (Retrieval-Augmented Generation) в Personal Assistant Bot.

## 🎯 Что такое RAG?

RAG позволяет боту отвечать на вопросы на основе ваших документов:
- PDF файлы
- Текстовые файлы
- Markdown документы

Бот создает векторные embeddings из документов и ищет релевантную информацию для ответов.

## 📥 Способы загрузки документов

### Способ 1: Ручная загрузка (Рекомендуется)

#### Шаг 1: Подготовьте документы

Поддерживаемые форматы:
- ✅ PDF (.pdf)
- ✅ Text (.txt)
- ✅ Markdown (.md)

#### Шаг 2: Скопируйте в папку

```bash
# Windows
copy my_document.pdf data\documents\
copy notes.txt data\documents\

# Linux/Mac
cp my_document.pdf data/documents/
cp notes.txt data/documents/
```

Или вручную через проводник:
```
Откройте папку: data/documents/
Скопируйте туда ваши файлы
```

#### Шаг 3: Перезапустите бота

```bash
# Остановите бота (Ctrl+C в терминале)
# Запустите снова
python main.py
```

При запуске бот:
1. Найдет все документы в `data/documents/`
2. Прочитает и разобьет их на фрагменты
3. Создаст embeddings через OpenAI
4. Сохранит в векторную базу ChromaDB

Вы увидите в логах:
```
Found 3 documents, indexing...
Loaded 42 chunks from my_document.pdf
Indexed 42 document chunks
```

#### Шаг 4: Проверьте индексацию

В Telegram:
```
/stats
```

Ответ:
```
📊 Статистика базы знаний

📄 Документов в индексе: 42
💾 Директория: C:\...\data\chroma_db

✅ База знаний готова к использованию!
```

## 🔍 Как работает RAG

### 1. Индексация (при загрузке)

```
Документ.pdf
    ↓
Извлечение текста
    ↓
Разбиение на фрагменты (chunks)
  ├─ Размер: 1000 символов
  └─ Пересечение: 200 символов
    ↓
Создание embeddings (OpenAI)
    ↓
Сохранение в ChromaDB
```

### 2. Запрос (при вопросе)

```
Ваш вопрос
    ↓
Создание embedding вопроса
    ↓
Поиск похожих фрагментов (Top-3)
    ↓
Формирование контекста
    ↓
GPT-4o генерирует ответ с контекстом
    ↓
Ответ с указанием источников
```

## ⚙️ Настройки RAG

В файле `config.py` можно настроить:

```python
# Размер фрагментов документа
RAG_CHUNK_SIZE = 1000

# Пересечение между фрагментами
RAG_CHUNK_OVERLAP = 200

# Количество релевантных фрагментов для контекста
RAG_TOP_K = 3
```

## 🎯 Лучшие практики

### 1. Подготовка документов

✅ **Хорошо:**
- Чистый, структурированный текст
- Заголовки и разделы
- Четкие формулировки

❌ **Плохо:**
- Сканированные изображения (OCR не поддерживается)
- Таблицы со сложной структурой
- Неструктурированный текст

### 2. Организация документов

Рекомендуется:
```
data/documents/
├── projects/
│   ├── project_a.pdf
│   └── project_b.pdf
├── contracts/
│   └── contract_2024.pdf
└── documentation/
    ├── api_docs.md
    └── user_guide.txt
```

### 3. Формулировка вопросов

✅ **Хорошие вопросы:**
- "Найди информацию о бюджете проекта X"
- "Какие требования указаны в документе?"
- "Суммируй основные пункты договора"

❌ **Плохие вопросы:**
- "Что там про деньги?" (слишком расплывчато)
- "Скажи всё про проект" (слишком широко)

**Удалить все и переиндексировать:**
```bash

rmdir /s data\chroma_db  # Windows

# Удалить ненужные документы из data/documents/

# Перезапустить бота (создаст новый индекс)

```

## ❓ Troubleshooting

### Проблема: Бот не находит документы

**Решение:**
1. Проверьте путь: `data/documents/`
2. Проверьте формат файлов (PDF, TXT, MD)
3. Перезапустите бота

### Проблема: Ошибка при индексации

**Решение:**
1. Проверьте логи: `bot.log`
2. Убедитесь что OpenAI API ключ валиден
3. Проверьте что файл не поврежден

### Проблема: Бот отвечает не по теме

**Возможные причины:**
- Документы не содержат нужной информации
- Вопрос сформулирован нечетко
- Недостаточно релевантных фрагментов

**Решение:**
- Переформулируйте вопрос более конкретно
- Добавьте больше документов по теме
- Увеличьте `RAG_TOP_K` в config.py

### Проблема: "База знаний пуста"

**Решение:**
1. Добавьте документы в `data/documents/`
2. Перезапустите бота
3. Проверьте `/stats`

## 📈 Производительность

### Скорость индексации

- PDF (10 страниц): ~5-10 секунд
- TXT (100KB): ~2-3 секунды
- Зависит от размера и OpenAI API

### Время ответа

- Поиск в базе: <1 секунда
- Генерация ответа: 2-5 секунд
- Общее время: 3-6 секунд

### Ограничения

- Максимальный размер файла: 20 MB (через бота)
- Рекомендуемое количество документов: до 1000
- Размер векторной БД: зависит от количества фрагментов

## 🔒 Приватность

- Документы хранятся локально в `data/documents/`
- Embeddings в `data/chroma_db/`
- OpenAI видит только текст при создании embeddings
- Документы не покидают ваш сервер
